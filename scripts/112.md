이번 영상에서는 Groovy로 작성된 `build.gradle` 파일에 KTS를 적용하는 법에 대해 알아보도록 하겠습니다.

현재 Android Studio에서 새로 작성하는 프로젝트의 `build.gradle`은 기본적으로 Groovy-DSL로 만들어지기 때문에 KTS를 적용하기 위해서는 수동으로 스크립트를 변환하여야 합니다. 여기서는 지금까지 만들어 온 프로젝트의 스크립트를 변환하면서 사용법을 알아보도록 하겠습니다.

## build.gradle 변환

### ' 를 "로 변환

Groovy는 `'`와 `"`를 모두 사용가능하지만 KTS는 `"`만을 허용하기 때문에 우선은 ' 를 "로 변환합니다. 일괄변환 기능을 사용하면 됩니다.

### ()를 붙임

다음은 함수로 처리되어야 하는 부분에 `()`를 붙여줍니다.

- `plugins`블럭 내부의 `id`
- `proguardFiles`
- `dependencies`블럭 내부의 `implementation` 등

### =를 붙임

프로퍼티로 처리되어야 하는 부분에 `=`를 붙여줍니다. 바꾸는 김에 [자바 11의 기능을 사용](https://developer.android.com/studio/releases/gradle-plugin#java-11)할 수 있도록 컴파일에 사용할 자바 소스코드 버전도 11로 올려 줍니다. 여기까지는 Groovy인 상태로도 빌드에 성공합니다.

- `compileSdk`
- `applicationId`
- `minSdk`
- `targetSdk`
- `versionCode`
- `versionName`
- `testInstrumentationRunner`
- `sourceCompatibility`
- `targetCompatibility`
- `buildFeatures`블럭 내부의 `viewBinding`
- `kapt`블럭 내부의 `correctErrorTypes`

### 파일명 변경

이 상태에서 두 `build.gradle`의 파일 이름을 `build.gradle.kts`로 변경하고 빌드하면 빌드가 실패합니다. 일단은 KTS를 사용할 수 있게 하기 위해 오류가 나오는 부분은 주석처리하고 빌드를 성공시키도록 합니다.

### 빌드 실패부분 처리

이제 빌드가 실패하는 부분을 수정합니다.

```diff
android {
  buildTypes {
+    named("release") {
-      minifyEnabled false
+      isMinifyEnabled = false
    }
  }
}
```

Project level의 커스텀 태스크는 다음과 같이 수정합니다.

```diff
- task clean(type: Delete) {
-     delete rootProject.buildDir
- }

+ tasks.register<Delete>("clean") {
+     delete(rootProject.buildDir)
+ }
```

## settings.gradle 변환

다음은 settings.gradle의 내용을 수정하고 settings.gradle.kts로 변환한 뒤 빌드를 수행합니다.

```diff
-include ':app'
+include(":app")
```

이제 모든 gradle 파일에 KTS가 적용되었습니다.

## buildSrc 도입

현재는 `app/build.gradle.kts`에 모든 디펜던시 정보를 기입하고 있습니다만 `buildSrc` 디렉토리를 사용하면 디펜던시 정보를 외부로 빼내어 따로 관리할 수 있습니다.

우선은 프로젝트 아래에 `buildSrc` 디렉토리를 만든 뒤 내부에 다음 내용의 `build.gradle.kts` 파일을 만들어 KTS를 활성화 한 뒤 싱크합니다. 그러면 프로젝트에서 `buildSrc` 디렉토리를 인식할 수 있게 됩니다.

```kotlin
repositories {
    mavenCentral()
}

plugins {
    `kotlin-dsl`
}
```

다음은 `buildSrc > src > main > java` 폴더를 생성합니다. 그리고 `java` 내부에 디펜던시 정보를 담은 `object` 클래스를 만들어 줍니다.

## 그 외의 수정

여기에서 다루지 않은 `build.gradle`의 더 많은 사용법에 대해서는 Gradle 공식 홈페이지에서 제공하는 [Migrating build logic from Groovy to Kotlin](https://docs.gradle.org/current/userguide/migrating_from_groovy_to_kotlin_dsl.html) 문서나, 구글에서 만든 공식 앱인 [The Google I/O Android App](https://github.com/google/iosched)을 보시면 도움이 될 것 같습니다.
